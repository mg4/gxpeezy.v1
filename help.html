Jefferson Parish Sheriff’s Office Investigative GIS Tool

The Jefferson Parish Sheriff’s Office Investigative GIS Tool (JPSO-IGT) provides JPSO law enforcement personnel relevant geospatial information that can be used to aid in investigative, criminal, tactical and strategic crime analysis. This software program allows authorized JPSO users to query and filter arrestee/inmate data to create relevant maps overlaid with prior incident data from the JPSO Case Management System (CMS).  By performing a geospatial query on a location, investigators can identify potential witnesses or informants.

The Jefferson Parish Sheriff’s Office Investigative GIS Tool Desktop

Click on the Jefferson Parish Sheriff’s Office Investigative GIS Tool (JPSO-IGT) icon to launch the application.  The JPSO-IGT application has a graphical user interface controlled using the mouse and keyboard.  The desktop features the Map Viewport with associated Options on the left, and any Query Results in the bottom section.

Options Menu

The Options Menu features two sections, Filters and Layers, to allow users to input data to search by or change layer selections.  The Options menu is expanded by default, but can be collapsed by clicking on the Options menu header or the button in the corner.  ­­­Collapsing the menu displays more of the map, while still allowing the user to mouse-over to open the Options menu for quick viewing.

 

Figure 2: Options menu header

 

Filters

The Filters section provides search parameters to enter information to search by to provide relative spatial data from the JPSO CMS database.  The Filter by date parameters offer the user Start Date and End Date fields to enter date and time ranges.  The Filter by radius parameters allows the user to enter an Address or select a Point on the map with a click of the mouse, and the Radius to determine the area to search within.  The radius default is set at 2 miles, but can be adjusted as needed.  The Filter by attribute parameters provide the user attribute fields to search arrestee/inmate data (ex. Name, Sex, Date of Birth, Race, Address, etc.).  Attributes are inclusive, meaning all data entered in attribute fields is used to search by; blank fields are not included in the search.

 

Figure 3: Filters section search parameters

 

Filter Functions

Filter Functions at the bottom of the Filters section allow the user to “Query” the JPSO CMS database or “Clear” the fields in the search parameters. 

 

Figure 4: Filter Functions

 

To perform a search in the JPSO-IGT application:

1.       In the Options menu, make sure the Filters section is open.

2.       Enter data in the search parameter fields:

a.       Filter by date:

Start Date: type a date or click the calendar icon to open the calendar control feature to select a date, and select a time to start search from. 

End Date: type a date or click the calendar icon to open the calendar control feature to select a date, and select a time to end search on. 

b.      Filter by radius:

Radius: enter a range for the search area from the address entered or point selected.

Address: enter an address and press the Enter key to set center of search area.

 -  OR  -

Point: click the “Point” button and use mouse to set center of search area on the map.

c.       Filter by attribute:

First Name: enter the first name of the arrestee or inmate.

Last Name: enter the last name of the arrestee or inmate.

Date of Birth: enter the date of birth of an arrestee or inmate.

Sex: select the gender of the arrestee or inmate.

Race: select the race of the arrestee or inmate.

Street No.: enter the street number of the arrestee or inmate.

Street: enter the street name of the arrestee or inmate.

City: enter the city of the arrestee or inmate.

State: enter the state of the arrestee or inmate.

CCN: enter the unique CCN identifying number of the arrestee or inmate.

ATN: enter the unique ATN identifying number of the arrestee or inmate.

3.       Click a Filter Function:

a.       “Query” - click to search the JPSO CMS database

b.      “Clear” - click to delete data in the search parameters.

 

 

Map

The Map is a graphical representation of data and layers.  Map content is managed by the data entered in the Filters search parameters and displays the area within the radius chosen of the address entered or point selected.  Users can hold down the left mouse button to “pan” the map to adjust the visible area or can use the arrows at the top left of the Map to move the map up, down, left or right.  A zoom scale is provided to adjust the zoom level of the Map, or users can use the mouse scroll wheel to zoom in/out.

Layers

The Layers section provides Overlays and Base Layers options representing different geo-referenced spatial data on the map.  Visibility of each layer is managed by selecting the checkbox next to a layer.

The default map Overlays are Parish Boundaries which shows the borders of the parishes within Louisiana, and ARMMS (Automated Record Management and Mapping System) which is a records management system for law enforcement agencies.  Both of the default Overlays can be applied to the map, or the user can choose to select only one to apply only the selected overlay to the map. 

The default map Base Layer is Google Roadmap which displays the normal, default 2D road map view from Google Maps; however the Base Layer can be changed to either Google Satellite which displays satellite images from Google Earth, Google Hybrid which displays a mix of satellite and roadmaps, or Google Terrain which displays a physical map showing terrain information.

Query Results

After a Filters search parameter query is performed, the Query Results table provides records from the JPSO CMS database for incidents located within the area of the Filter search parameters queried.  Each query result record is a “feature” and, based on the Overlays applied, has “attributes” that provide detailed information for that incident record. 

 

Figure 7: Query Result from a Filters query

 

Clicking on a record in the Query Results highlights the selected “feature” on the map. 

Figure 8: Selected Query Result "feature" highlighted as a point on the map

The user can use buttons in the Toolbar to perform additional map functions for the selected “feature”: 

·         Zoom to Selected Feature – zooms the map in to the selected arrest/inmate “feature”

·         Get Feature Info – displays data “attributes” associated with the selected “feature”

Figure 9: Get Feature Info for selected "feature"

 

Export to CSV

After performing a query, the data returned in the Query Results section can be saved for further analysis.  By clicking the Export to CSV button, the JPSO-IGT application exports the query results data to a Microsoft Excel Comma Separated Values file which can be saved by the user.  The saved CSV file can then be imported into Microsoft Excel for further manipulation.
